# 什么是双链笔记

> 双链（Bidirectional Link）是笔记之间的双向连接。当笔记 A 链接到笔记 B 时，B 会自动显示来自 A 的反向引用。

## 为什么双链重要

传统的文件夹和标签是**单向分类**——一篇笔记只能属于一个文件夹，或打上几个标签。双链打破了这种限制：

- **网状结构** — 知识不是树状的，而是相互关联的网
- **意外发现** — 反向链接让你发现意想不到的关联
- **渐进构建** — 不需要预先设计完美的分类体系

## 在 Obsidian 中使用双链

### 基本语法

```
这是一条指向 [[另一篇笔记]] 的链接
```

### 链接到标题

```
[[笔记名#某个标题]]
```

### 自定义显示文本

```
[[笔记名|点这里查看]]
```

### 嵌入笔记内容

```
![[另一篇笔记]]
```

## 与传统方式的对比

| 方式 | 示例 | 局限 |
|------|------|------|
| 文件夹 | `笔记/技术/` | 一篇笔记只能在一个文件夹 |
| 标签 | `#技术 #云原生` | 只有标签层面，没有上下文 |
| **双链** | `[[云原生]]` | 链接即关系，自动反向引用 |

## 实践建议

1. **随手链接** — 写笔记时遇到相关概念就 `[[链接]]` 过去，不要犹豫
2. **创建 MOC** — 用 [[MOC/MOC - 技术架构\|MOC - 技术架构]] 这样的笔记作为知识入口
3. **定期回顾** — 查看反向链接，发现新的关联
4. **原子笔记** — 一篇笔记讲清楚一个概念，便于复用和链接
